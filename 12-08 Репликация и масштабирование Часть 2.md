### Задание 1

Опишите основные преимущества использования масштабирования методами:

- активный master-сервер и пассивный репликационный slave-сервер; 
- master-сервер и несколько slave-серверов;


*Дайте ответ в свободной форме.*

### Решение 1

- Активный master-сервер и пассивный slave-сервер
Slave-сервер хранит копию данных с master-сервера. Это удобно тем, что данные не потеряются при сбое основного сервера. Также часть запросов на чтение можно отправлять на slave, чтобы снизить нагрузку на master. Настраивается такая схема довольно просто.

- Master-сервер и несколько slave-серверов
Такая схема помогает справляться с большой нагрузкой. Master отвечает за запись данных, а slave-серверы — за чтение. Если один slave выйдет из строя, остальные продолжат работать, поэтому система становится более надежной.

- DRBD (Distributed Replicated Block Device)
DRBD делает копию данных между двумя серверами. Если основной сервер ломается, можно быстро переключиться на резервный сервер и продолжить работу почти без потери данных.

- SAN-кластер
SAN-кластер используется для надежного хранения данных. У него высокая производительность и хорошая отказоустойчивость, потому что данные хранятся сразу на нескольких устройствах. Еще плюс в том, что оборудование можно обновлять или менять без остановки всей системы.

### Задание 2


Разработайте план для выполнения горизонтального и вертикального шаринга базы данных. База данных состоит из трёх таблиц: 

- пользователи, 
- книги, 
- магазины (столбцы произвольно). 

Опишите принципы построения системы и их разграничение или разбивку между базами данных.

*Пришлите блоксхему, где и что будет располагаться. Опишите, в каких режимах будут работать сервера.* 

### Решение 2

С помощью вертикального шардинга общую базу данных можно разделить на отдельные базы (по назначению):
- Books (id, title, author, genre, price)
- Stores (id, address, store_name, phone_number)
- Customers (id, name, email, password)

Принцип работы

Каждая база отвечает только за свои данные.
Это уменьшает нагрузку и упрощает обслуживание системы.

При горизонтальном шардинге данные одной таблицы делятся между несколькими серверами.

Пример для таблицы пользователей:
- Server 1 - пользователи с ID 1–100000
- Server 2 - пользователи с ID 100001–200000
- Server 3 -пользователи с ID 200001–300000

То же самое можно сделать для книг и магазинов.
