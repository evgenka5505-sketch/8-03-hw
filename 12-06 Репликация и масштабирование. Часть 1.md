## Задание 1. 

На лекции рассматривались режимы репликации master-slave, master-master, опишите их различия.

## Решение 1

Репликация типа master-slave предполагает наличие одного основного сервера (master) и одного или нескольких зависимых серверов (slave). Все операции записи и изменения данных выполняются только на master-сервере, а slave-серверы получают и копируют эти изменения для поддержания актуальной копии базы данных. Обычно такой режим используется для распределения нагрузки на чтение и повышения отказоустойчивости.

В режиме master-master оба сервера одновременно являются и основными, и подчинёнными по отношению друг к другу. Каждый из них может принимать операции записи и изменения данных, а внесённые изменения автоматически синхронизируются между серверами. Такой подход позволяет повысить доступность системы и распределить нагрузку между несколькими серверами.

## Задание 2

Выполните конфигурацию master-slave репликации, примером можно пользоваться из лекции.

Приложите скриншоты конфигурации, выполнения работы: состояния и режимы работы серверов.

## Решение 2

Создание docker и запуск master и slave

```bash
docker network create mysql-repl
docker run -d \
--name mysql-master \
--network mysql-repl \
-p 3306:3306 \
-e MYSQL_ROOT_PASSWORD=root123 \
mysql:8.0 \
--server-id=1 \
--log-bin=mysql-bin \
--binlog-format=ROW
docker run -d \
--name mysql-slave \
--network mysql-repl \
-p 3307:3306 \
-e MYSQL_ROOT_PASSWORD=root123 \
mysql:8.0 \
--server-id=2 \
--relay-log=relay-bin
```

Подключение к master и создание пользователя репликации

```bash
docker exec -it mysql-master mysql -u root -p
CREATE USER 'Evgen'@'%' IDENTIFIED WITH mysql_native_password BY '0000';
GRANT REPLICATION SLAVE ON *.* TO 'Evgen'@'%';
FLUSH PRIVILEGES;
SHOW MASTER STATUS;
```

Подключение к Slave, настройка подключения к Master и запуск

```bash
docker exec -it mysql-slave mysql -u root -p
CHANGE MASTER TO
  MASTER_HOST='mysql-master',
  MASTER_USER='Evgen',
  MASTER_PASSWORD='0000',
  MASTER_LOG_FILE='mysql-bin.000003',  
  MASTER_LOG_POS=1060;    
  MASTER_SSL=0; 
START SLAVE;
SHOW SLAVE STATUS\G
```
